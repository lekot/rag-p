# RAG Platform — краткий обзор

Retrieval-Augmented Generation (RAG) — это архитектура, в которой языковая модель получает контекст из внешнего хранилища знаний перед генерацией ответа. Платформа предоставляет конвейер: загрузка документов, чанкинг, эмбеддинги, индекс в pgvector, ретривер, реранкер и генератор.

Основные сущности:

- **Organization** — изолированный tenant, владеет всеми данными.
- **Dataset** — набор загруженных документов и golden Q&A.
- **Pipeline** — сохранённая конфигурация плагинов (chunker → retriever → generator).
- **Experiment** — grid-search по комбинациям плагинов с автоматической оценкой.
- **Run** — одно исполнение pipeline на одном вопросе.

Точка входа клиента: загрузить корпус → создать API key → вызвать `POST /api/v1/rag/query`. Ответ содержит сгенерированный текст, использованные чанки и трассу пайплайна.
