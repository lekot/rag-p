# Architecture Audit — 2026-05-05

## Logical gaps and inconsistencies found during codebase review

### 1. DELETE endpoints absent for experiments and pipelines

| Resource | API | Frontend router | Frontend page | Status |
|----------|-----|-----------------|---------------|--------|
| Datasets | `DELETE /datasets/{id}` | `datasets.delete` | no delete button | ⚠️ |
| Experiments | **missing** | **missing** | **missing** | 🔴 |
| Pipelines | **missing** | **missing** | **missing** | 🔴 |
| Documents | `DELETE /datasets/{id}/documents/{id}` | `datasets.deleteDocument` | — | ✅ |
| API Keys | `DELETE /keys/{id}` | `keys.revoke` | — | ✅ |

Experiments and pipelines accumulate forever — no way to clean them up from API or UI.

**Fix**: Add `DELETE /experiments/{id}` and `DELETE /pipelines/{id}` with cascade on PipelineVersions + Runs.

### 2. Auth gaps

| Endpoint | Auth check | Risk |
|----------|-----------|------|
| `GET /experiments` | **none** — no `require_scope` / `require_organization` | Anyone can list all experiments in the DB |
| `GET /experiments/{id}` | **none** | Anyone can read any experiment |
| `POST /pipelines/{id}/runs` | **none** | Anyone who knows a pipeline ID can execute it |

Compare with `POST /datasets/{id}/ask` which has `require_organization` — inconsistent.

**Fix**: Add `require_organization` or `require_scope("read")` to experiment GET endpoints.
Add org-scoping to `POST /pipelines/{id}/runs`.

### 3. Quota bypass via `POST /pipelines/{id}/runs`

`POST /pipelines/{id}/runs` (routes_runs.py) executes the pipeline **without** calling:
- `_reserve_dataset_query_quota` / `consume_q`
- `get_active_subscription` check

`POST /datasets/{id}/ask` with pipeline_id (routes_datasets.py) **does** check quotas.
`POST /rag/query` with pipeline_id (routes_rag.py) **does** check quotas.

This means the runs endpoint is a free LLM access path — no subscription required.

**Fix**: Add `_reserve_dataset_query_quota` or `consume_q` call to `create_run`.

### 4. Missing org validation in `ask_dataset` pipeline path

```python
# routes_datasets.py ask_dataset — pipeline path:
pl = pl_result.scalar_one_or_none()
if pl is not None and pl.current_version_id:
    ...  # NO check: pl.organization_id == organization_id
```

Compare with routes_rag.py which validates:
```python
if pl is None or pl.organization_id != org.id or pl.dataset_id != body.dataset_id:
    raise HTTPException(...)
```

**Risk**: A user in org A can use `ask_dataset` with a pipeline_id from org B (if they know the UUID).

**Fix**: Add `pl.organization_id == organization_id` check.

### 5. Experiment watchdog race condition during chunk re-embedding

`_ensure_chunks_embedded` can take 30+ seconds (OpenAI) to minutes (Ollama) to re-embed all chunks.
The watchdog cron (`mark_stale_experiments_failed`) runs every 2 minutes and force-fails
experiments whose `updated_at` is older than `experiment_stale_timeout_seconds` (900s).

`_touch()` (heartbeat) is called between combos, but **not** during `_golden_metrics` /
`_self_test_metric` where re-embedding happens.

With Ollama (~7s per chunk, sequential API), re-embedding 1552 chunks would take ~3 hours —
way past the 15-minute watchdog timeout AND the worker job timeout (600s).

**Fix**: Call `_touch()` inside the batch loop of `_ensure_chunks_embedded`.

### 6. Runs endpoint lacks pagination

```typescript
// apps/web/src/server/routers/runs.ts
const raw = await apiClient.get<RunListItem[]>(
  `/api/v1/runs?organization_id=${ctx.organization_id}`
);
```

Also in the API:
```python
# routes_runs.py list_runs
stmt = select(Run).where(Run.organization_id == organization_id)
```

No `LIMIT` / `OFFSET` — loads ALL runs for the org into memory. Breaks at ~1000+ runs.

**Fix**: Add `limit` (default 50) and `offset` query params to the API endpoint.
Add pagination state to the frontend runs page.

---

### 7. Self-test metric uses statistically insignificant sample

```python
_SELF_TEST_SAMPLE_LIMIT = 5
```

The self-test (used when no golden Q&A exists) evaluates only **5 chunks**.
Hit rate on 5 samples has enormous variance — 0.0 or 1.0 are common by chance.
This makes the experiment leaderboard meaningless for datasets without golden items.

**Fix**: Increase sample size, or skip self-test entirely and require golden Q&A.

### 9. Chunk re-embedding: newly fixed architecture

```
before (broken):
  _rechunk_documents_for_slot → chunks with embedding=NULL ❌
  _golden_metrics              → dense branch finds nothing
    
after (fixed in this audit):
  _rechunk_documents_for_slot → chunks with embedding=NULL (only chunking)
  _ensure_chunks_embedded     → checks embedder.dim vs stored vector_dims()
                               → re-embeds if mismatch
  _golden_metrics / _self_test → dense branch works ✅
```
